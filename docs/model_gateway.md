# XAH Model Gateway

The project can call XAH directly without running a separate LiteLLM or 9Router
proxy. XAH exposes protocol-compatible endpoints for the project and common
coding agents.

## Project configuration

Set the OpenAI-compatible SDK variables in `.env`:

```dotenv
OPENAI_API_KEY="your-xah-api-key"
OPENAI_BASE_URL="https://api.xah.io/v1"
CHAT_MODE="auto"
CHAT_MODEL="gpt-5.4"
```

The LLM and remote embedding clients both read these variables. Secrets are not
stored in `configs/config.yaml` or serialized into ChromaDB embedding metadata.

`CHAT_MODE=auto` keeps RAG optional. The API uses the local news collection only
when ChromaDB is installed and `data/chroma_db` exists; otherwise it sends the
conversation directly to the configured model gateway. `CHAT_MODE=direct`
always skips retrieval. `CHAT_MODE=rag` requests retrieval but still falls back
to direct chat when the local RAG pipeline cannot initialize or has no data.

## Test the chat UI without RAG

Create `.env` from `.env.example`, set a valid XAH key and chat model, then run:

```powershell
# Terminal 1
uvicorn api.main:app --reload --port 8000

# Terminal 2
Set-Location ui
npm install
npm run dev
```

Open `http://localhost:5173` and sign in with `analyst` / `pass123`. No ChromaDB
installation or ingestion step is required for direct chat. The backend imports
retrieval dependencies lazily, so a missing RAG environment does not prevent the
API or UI from starting.

The default project embedding remains local `all-MiniLM-L6-v2` with 384 output
dimensions. To opt into a remote embedding model, update the embedding block:

```yaml
embedding:
  provider: "openai"
  model_name: "<xah-embedding-model-id>"
  dimensions: <exact-output-dimension>
```

Changing the embedding model or dimensions requires rebuilding the ChromaDB
collection. A collection must not contain vectors generated by different models
or with different dimensions.

## Endpoint matrix

| Client or protocol | Endpoint |
|---|---|
| OpenAI Chat Completions | `https://api.xah.io/v1/chat/completions` |
| OpenAI Responses / Codex | `https://api.xah.io/v1/responses` |
| Anthropic Messages / Claude Code | `https://api.xah.io/v1/messages` |
| OpenAI Embeddings | `https://api.xah.io/v1/embeddings` |
| Gemini generateContent | `https://api.xah.io/v1beta/models/<model>:generateContent` |
| Ollama Chat | `https://api.xah.io/api/chat` |

## Codex CLI

Store the XAH secret in `XAH_API_KEY`, then add a provider and profile to
`~/.codex/config.toml`:

```toml
[model_providers.xah]
name = "XAH"
base_url = "https://api.xah.io/v1"
env_key = "XAH_API_KEY"
wire_api = "responses"

[profiles.xah]
model_provider = "xah"
model = "<xah-codex-model-id>"
```

Run Codex with `codex --profile xah`.

## Claude Code

Claude Code appends `/v1/messages`, so its base URL must not include `/v1`:

```powershell
$env:ANTHROPIC_BASE_URL="https://api.xah.io"
$env:ANTHROPIC_AUTH_TOKEN=$env:XAH_API_KEY
claude
```

## OpenCode

Use the OpenAI provider package for models served through the Responses API:

```json
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "xah": {
      "npm": "@ai-sdk/openai",
      "name": "XAH",
      "options": {
        "baseURL": "https://api.xah.io/v1",
        "apiKey": "{env:XAH_API_KEY}"
      },
      "models": {
        "<xah-model-id>": {
          "name": "XAH Model"
        }
      }
    }
  }
}
```

For a model that only supports `/v1/chat/completions`, replace the package with
`@ai-sdk/openai-compatible`.

## Gateway smoke test

The smoke test always checks `/v1/models` and checks other endpoints only when
their model arguments are supplied. It never prints credentials, prompts, or
model response content.

```powershell
python scripts/check_model_gateway.py `
  --chat-model "<chat-model-id>" `
  --responses-model "<responses-model-id>" `
  --messages-model "<anthropic-model-id>" `
  --embedding-model "<embedding-model-id>" `
  --embedding-dimensions 1536
```

Add `--stream` to validate SSE transport for Chat Completions, Responses, and
Messages. Gemini and Ollama checks are available through `--gemini-model` and
`--ollama-model`.

# Final NewsQA Evaluation Output

## Dataset identity

The finalized benchmark is `newsqa_200_11064`, schema version `2.1`. It uses
NewsQA revision `728e52920b8e4ffcfaad93fa47556f26a1d82546` with seed `42`.

| Component | Count |
| --- | ---: |
| Evaluation articles | 200 |
| Retrieval-only distractor articles | 10,864 |
| Total corpus articles | 11,064 |
| Raw NewsQA questions | 1,340 |
| Scored questions | 1,336 |
| Clarified question pairs | 1,078 |
| Excluded questions | 4 |
| Corrected answers | 298 |
| Corpus chunks | 19,263 |

All final files are under:

```text
data/evaluation/newsqa_200_11064/final/
```

The primary scored test set is `testset_reviewed_original.jsonl`. The
`testset_resolved.jsonl` file is its same-size clarification treatment. Do not
concatenate these files because that would count the same source questions
twice.

## Output files

```text
final/
```

### Question datasets

| File | Rows | Purpose |
| --- | ---: | --- |
| `testset_original.jsonl` | 1,340 | Immutable extraction of every selected NewsQA question. It preserves raw wording, answers, and evidence and includes the four questions later excluded. Use for provenance and raw-baseline diagnostics, not as the primary reported score. |
| `testset_reviewed_original.jsonl` | 1,336 | Primary benchmark. It keeps each original question wording while applying reviewed answers and evidence. Excluded questions are absent. |
| `testset_resolved.jsonl` | 1,336 | Clarification treatment. It has exactly one row for every primary row: 1,078 use approved clarified wording and 258 retain the original wording. Answers, evidence, and corpus are identical to the primary benchmark. |
| `testset_clarified.jsonl` | 1,078 | Paired subset containing only approved clarifications. Join it to the primary set through `source_question_id` to measure the effect of clarification on the same questions. |
| `excluded_questions.jsonl` | 4 | Questions removed from scoring because no defensible article-grounded gold target remained. It records exclusion reasons and review notes so exclusions are reportable. |

The required count relationships are:

```text
testset_original = review_annotations = 1,340
testset_reviewed_original = testset_resolved = 1,340 - 4 = 1,336
testset_clarified = human_non_standalone scored rows = 1,078
```

### Review and retrieval artifacts

| File | Contents |
| --- | --- |
| `review_annotations.jsonl` | One final review record for every raw question, including excluded rows. It contains proposal provenance, human decision, corrections, clarifications, supporting quotes, and exclusion state. This is the audit-oriented annotation table rather than a benchmark input. |
| `chunks.jsonl` | All 19,263 chunks produced from the 11,064-article corpus with the production recursive chunker (`512` tokens, `64` overlap). These IDs are the retrieval relevance keys used by the question files. |
| `bm25.pkl` | Serialized BM25 index over exactly the chunks in `chunks.jsonl`. Load only as a trusted project artifact and never pair it with another chunk file. |
| `integrity_report.json` | Final counts and readiness checks. A usable finalized dataset has `status: "passed"`, `phase: "review_complete"`, zero pending/adjudication decisions, and equal `chunks` and `indexed_chunks`. |

The dense index is stored separately in `data/chroma_db` because Chroma is a
persistent database rather than a JSONL artifact. Its finalized collection is:

```text
newsqa_val200_s42_7e16e8_66785e
```

It contains 19,263 vectors generated by `all-MiniLM-L6-v2` with 384
dimensions. The BM25 index, Chroma collection, and `chunks.jsonl` represent the
same chunk population.

## Question row schema

Fields appear as applicable; raw rows intentionally omit reviewed fields.

| Field | Meaning |
| --- | --- |
| `question_id` | Stable NewsQA question ID. In `testset_clarified.jsonl` only, `::clarified` is appended to distinguish the paired variant. |
| `source_question_id` | Original stable question ID used to join reviewed, resolved, clarified, excluded, and annotation records. |
| `question` | Wording evaluated for this row. |
| `question_variant` | `original` or `clarified`. In the resolved set this identifies whether clarification was applied. |
| `article_key` | Canonical `newsqa_*` ID of the source evaluation article. |
| `ground_truth` | Primary gold answer. Reviewed files contain the approved answer, which may differ from the raw NewsQA answer. |
| `accepted_answers` | Accepted gold strings for answer matching; the primary `ground_truth` is always included. |
| `evidence` | Primary evidence-span text. |
| `evidence_spans` | Reviewed `[start, end)` character offsets into the complete source article. The stored text must exactly match that slice. |
| `source_ground_truth` | Original NewsQA answer when an answer was corrected. |
| `source_evidence_spans` | Original NewsQA evidence offsets when reviewed evidence differs. |
| `answer_modified` | Whether review changed the answer or evidence used for scoring. |
| `answer_review_notes` | Human-readable justification for an answer/evidence correction. |
| `article_chunk_ids` | Every chunk belonging to the source article. |
| `relevant_chunk_ids` | Chunks overlapping the reviewed evidence spans. Retrieval metrics compare returned chunk IDs against this list. |
| `standalone_label` | Final state: `human_standalone`, `human_non_standalone`, or `human_excluded`. |
| `ambiguity_reasons` | Review issue codes associated with the question or answer. |
| `excluded` | `true` only in the exclusion artifact. |
| `exclusion_reasons` | Allowed reasons supporting exclusion from scored variants. |
| `review_notes` | Human explanation associated with an exclusion or review decision. |

## Chunk row schema

Each line of `chunks.jsonl` contains:

| Field | Meaning |
| --- | --- |
| `id` | Stable chunk ID, such as `85a6d026fc61_chunk_0`. Question relevance fields reference this value. |
| `text` | Exact chunk text embedded and indexed for retrieval. |
| `metadata.article_id` | The chunker's short article identifier used in chunk IDs. |
| `metadata.canonical_article_id` | Full `newsqa_*` article ID used by corpus and question artifacts. |
| `metadata.chunk_index` | Zero-based position within the article. |
| `metadata.corpus_role` | `evaluation` or `distractor`. |
| `metadata.dataset_split` | `validation` for evaluation articles or `train` for distractors. |
| `metadata.title`, `publisher`, `url`, `author`, `publish_date` | Source metadata available to retrieval and citation formatting. Empty strings indicate unavailable source values. |

## Manifests and provenance

Two tracked manifests make the generated data reproducible and prevent index
mismatches:

| Manifest | Role |
| --- | --- |
| `evaluation/manifests/newsqa_200_11064.selection.json` | Records dataset revision, seed, selected IDs, scan/filter statistics, artifact hashes, and the 200/10,864 corpus composition. |
| `evaluation/manifests/newsqa_200_11064.variant.json` | Records every final artifact hash, pipeline configuration hash, chunking and embedding settings, Chroma path/collection/count, and final integrity statistics. Its final status is `review_complete`. |

The staging directory remains the provenance source for the final output:

```text
staging/corpus/evaluation_articles.jsonl
staging/corpus/distractor_articles.jsonl
staging/questions/original_questions.jsonl
staging/review/review_queue_readable.json
staging/review/manifest.json
```

The migrated review manifest points back to the original proposal and human
approval audits. Staging files should not be supplied directly to the benchmark
runner.

## Running the benchmark

Use the variant manifest on every run. Its preflight validates the test-set
hash, config hash, collection, chunk file, and BM25 artifact before evaluation.

Primary retrieval benchmark:

```bash
.venv/bin/python scripts/run_benchmark.py \
  --retriever hybrid \
  --testset data/evaluation/newsqa_200_11064/final/testset_reviewed_original.jsonl \
  --variant-manifest evaluation/manifests/newsqa_200_11064.variant.json \
  --report-dir reports/newsqa_200_11064/original
```

Clarification comparison:

```bash
.venv/bin/python scripts/run_benchmark.py \
  --retriever hybrid \
  --testset data/evaluation/newsqa_200_11064/final/testset_resolved.jsonl \
  --variant-manifest evaluation/manifests/newsqa_200_11064.variant.json \
  --report-dir reports/newsqa_200_11064/resolved
```

Report the primary and resolved results separately. Their paired difference
measures the effect of clarification while holding answers, evidence, articles,
chunks, retrieval configuration, and question count constant.
